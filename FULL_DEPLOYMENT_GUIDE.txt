Step 1a: Download Armbian 26.2.1 for Orange Pi One

1. Go to: https://www.armbian.com/download/
   Or: https://redirect.armbian.com/orangepione/archive

2. Find: Armbian 26.2.1 for Orange Pi One
   
3. Download:
   • File: Armbian_26.2.1_Orangepione_trixie_current_6.12.74_minimal.img.xz

4. Verify download (optional but recommended):

Step 1b: Extract the Image

The downloaded file is compressed (.xz format)

macOS/Linux:
   $ cd ~/Downloads
   $ xz -d Armbian_26.2.1_Orangepione_trixie_current_6.12.74_minimal.img.xz
   
   (Creates: Armbian_26.2.1_Orangepione_trixie_current_6.12.74_minimal.img)

Windows:

   • Right-click .xz file → 7-Zip → Extract
   • Or use WinRAR, PeaZip, or similar

Step 1c: Prepare microSD Card
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

You need:
  • microSD card (16GB or larger recommended)
  • Card reader or USB adapter
  • Flashing tool

Linux (Using dd command):
   $ lsblk                                    # List disks
   $ sudo umount /dev/sdX*                    # Unmount if mounted
   $ sudo dd if=Armbian_26.2.1_*.img of=/dev/sdX bs=4M status=progress
   $ sync

   ⚠️  IMPORTANT: Replace /dev/sdX with your card device
   WARNING: Using wrong device will destroy data!

macOS (Using dd command):
   $ diskutil list                            # List disks
   $ diskutil unmountDisk /dev/diskX          # Unmount if mounted
   $ sudo dd if=Armbian_26.2.1_*.img of=/dev/rdiskX bs=4m
   $ sync

   ⚠️  Note: Use rdiskX (not diskX) for faster flashing

Windows (Using Balena Etcher - Recommended):
   1. Download: https://www.balena.io/etcher/
   2. Install and run Etcher
   3. Select image: Armbian_26.2.1_*.img
   4. Select drive: Your microSD card
   5. Click "Flash"
   6. Wait for completion

👉 After flashing:
   • Eject microSD card safely
   • Remove card from reader
   • Move to next step

PART 6: TROUBLESHOOTING WITH DIAGNOSTICS
════════════════════════════════════════════════════════════════════════════

If admin panel doesn't load, run the diagnostic tool:

On your Orange Pi:
   root@orangepione:~# cd Opensource_Centralize
   root@orangepione:~/Opensource_Centralize# sudo ./diagnose.sh

This will show:
   ✓ Port 5000 status (Flask)
   ✓ Port 80 status (Nginx)
   ✓ Firewall rules
   ✓ Process information
   ✓ Network connectivity
   ✓ Error logs
   ✓ Specific fix commands

Common Issues & Fixes:

Issue: Port 5000 not listening
Fix:   sudo systemctl restart pisonet
       sudo systemctl status pisonet

Issue: Port 80 blocked by firewall
Fix:   sudo ufw allow 80/tcp
       sudo ufw reload

Issue: Connection refused from network
Fix:   sudo ufw allow 5000/tcp
       sudo ufw allow 80/tcp
       sudo ufw reload

Issue: Can't find Orange Pi IP
Fix:   On Orange Pi: hostname -I
       Or check router's connected devices list

   root@orangepione:~# python3 -c "
   import gpiozero
