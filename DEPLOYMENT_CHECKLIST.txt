   │ $ sudo systemctl status pisonet                                 │

   ┌────────────────────────────────────────────────────────────────┐
   │ $ sudo systemctl status nginx                                   │

   ┌────────────────────────────────────────────────────────────────┐

💡 KEY POINTS TO REMEMBER:

Port 5000 - Flask Server:
   • This is where your application actually runs
   • Only on loopback: 127.0.0.1:5000
   • Must be LISTENING for system to work

Port 80 - Nginx HTTP:
   • This is what users access from their browsers
   • Forwards requests to Flask on 5000
   • Must be LISTENING and OPEN in firewall

Firewall Rules:
   • Without opening ports in firewall, even if services run, 
     clients can't reach them from other machines
   • setup.sh automatically opens ports with UFW
   • If still blocked, run: sudo ufw allow 80/tcp

Nginx Reverse Proxy:
   • Acts as middleman between clients and Flask
   • Listens on port 80
   • Forwards requests to Flask on 5000
   • Provides load balancing and caching
